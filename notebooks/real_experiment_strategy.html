Executive Summary

Running the full 12-circuit classical shadows benchmark on real quantum hardware requires careful planning due to queue times, job overhead, and the unique requirements of the shadows protocol. This document outlines practical strategies for obtaining publication-quality results within realistic time and resource constraints.

Key Insight: The actual QPU time for our entire benchmark (all circuits, all shots) would be only minutes. The bottleneck is everything else.

3. Classical Shadows: Hardware Challenges

Protocol-Specific Issue: Classical shadows require a different random measurement basis for each shadow. This means:
  • Each shadow = different circuit (random Cliffords appended)
  • 1,000 shadows = 1,000 unique circuits per base state
  • Cannot simply run "1,000 shots of the same circuit"

3.1 Shadow Count Requirements

Configuration Shadows per Circuit Total Shadows (12 circuits)
Current (10 rep × 1k shots) ~1,000 ~12,000
Publication (50 rep × 10k shots) ~10,000 ~120,000

3.2 Batching Strategies

To reduce job overhead, shadows can be batched:

  • Primitive batching: IBM's Sampler allows multiple circuits per job (up to ~300)
  • Session mode: Reserve QPU time to avoid repeated queue waits
  • Dynamic circuits: Some randomization possible within a single job

5. Recommended Experiment Strategy

Phase 1: Validation Run (1-2 days)
  • Circuits: 3 key circuits (S-GHZ-4, C-H2, M-PHASE-3)
  • Parameters: 10 replicates, 500 shadows each
  • Purpose: Validate pipeline, check noise levels, identify issues
  • Access: Free tier sufficient
Phase 2: Core Benchmark (3-5 days)
  • Circuits: 6 circuits (2 per family: S, C, O)
  • Parameters: 20 replicates, 1,000 shadows each
  • Purpose: Main publication data
  • Access: Premium recommended
Phase 3: Extended Analysis (Optional, 1 week)
  • Circuits: All 12 circuits
  • Parameters: 30 replicates, 2,000 shadows
  • Purpose: Comprehensive comparison, supplementary material
  • Access: Reserved session recommended

7. Specific Recommendations

For Maximum Impact with Limited Resources:

  1. Focus on 4-5 qubits: Best noise-to-signal ratio on current hardware
  2. Prioritize GHZ and chemistry circuits: Most compelling use cases
  3. Use 1,000-2,000 shadows: Sufficient for statistically significant results
  4. Run 20-30 replicates: Enables robust error bars
  5. Compare 2 protocols: Shadows vs. Direct Optimized (skip grouped baseline)
  6. Include noise characterization: Run calibration circuits alongside

Risk Mitigation
